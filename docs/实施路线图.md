# 实施路线图

> 从八字智能客服出发，分阶段演进为通用智能客服平台。

---

## 阶段概览

```
Phase 0 (当前)          Phase 1              Phase 2              Phase 3
八字单体应用     →    平台化重构       →    商业化 MVP      →    规模化运营
已完成                 4-6 周                4-6 周               持续迭代
                       ─────────             ─────────            ─────────
                       FastAPI 重构          多租户上线           行业拓展
                       插件体系              计费系统             渠道扩展
                       记忆升级              管理后台             AI Copilot
                       Widget 封装           官网上线             开放市场
```

---

## Phase 1：平台化重构（4-6 周）

**目标：** 将八字项目从单体应用重构为插件化平台，验证架构可行性。

### Week 1-2：后端骨架

| 任务 | 输出 | 优先级 |
|------|------|--------|
| FastAPI 项目初始化 + 目录结构 | 可运行的空项目 | P0 |
| PostgreSQL 数据模型（用户、会话、消息） | Alembic 迁移脚本 | P0 |
| 对话引擎核心（消息接收 → LLM调用 → 返回） | `/api/chat` 可用 | P0 |
| LLM Provider 抽象层（先接 DeepSeek） | 可切换模型 | P0 |
| SSE 流式输出 | 前端可接收流式 | P1 |

### Week 3-4：插件体系 + 记忆

| 任务 | 输出 | 优先级 |
|------|------|--------|
| 插件基类 + 加载器 | `BaseDomainPlugin` 可用 | P0 |
| 八字插件迁移（从旧项目提取） | 八字功能恢复 | P0 |
| 记忆系统重构（三层 → 多租户兼容） | 记忆 API 可用 | P0 |
| RAG 检索模块 | 知识库可查询 | P1 |
| 意图识别 + 槽位填充 | 可配置化 | P1 |

### Week 5-6：前端 Widget

| 任务 | 输出 | 优先级 |
|------|------|--------|
| Web Component 封装客服Widget | 一行JS嵌入 | P0 |
| Widget UI（对话界面、输入框、发送） | 基础可用 | P0 |
| Widget 配置化（颜色、位置、欢迎语） | 租户可定制 | P1 |
| 管理后台骨架（React/Vue） | 基础页面 | P1 |

**Phase 1 验收标准：**
- ✅ 八字客服在新架构上完整运行
- ✅ 能通过 Widget 嵌入任意网页
- ✅ 能通过修改 YAML 配置快速创建新领域插件

## Phase 2：商业化 MVP（4-6 周）

**目标：** 具备基本的商业运营能力，能接第一批付费客户。

### Week 7-8：多租户 + 计费

| 任务 | 输出 | 优先级 |
|------|------|--------|
| 租户注册/登录系统 | 注册流程可用 | P0 |
| 租户隔离（数据、配置、额度） | 租户间互不影响 | P0 |
| 计费引擎（对话计数、Token统计） | 用量可追踪 | P0 |
| 支付集成（微信支付/支付宝） | 可在线付款 | P0 |
| 套餐管理（免费/专业/企业） | 套餐可切换 | P1 |

### Week 9-10：管理后台

| 任务 | 输出 | 优先级 |
|------|------|--------|
| 租户控制台（配置、知识库、提示词） | 自助配置 | P0 |
| 对话记录查看/导出 | 可审计 | P0 |
| 数据看板（基础指标） | 可视化报表 | P1 |
| 知识库管理（上传/编辑/删除） | 文档管理 | P1 |
| 人工坐席后台（基础版） | 可接管对话 | P2 |

### Week 11-12：上线准备

| 任务 | 输出 | 优先级 |
|------|------|--------|
| Docker Compose 生产部署 | 一键部署 | P0 |
| 域名 + SSL + CDN | 可公网访问 | P0 |
| 官网/落地页 | 获客入口 | P0 |
| 文档（接入指南、API文档） | 客户自助 | P1 |
| 安全加固（SQL注入、XSS、限流） | 安全基线 | P0 |

**Phase 2 验收标准：**
- ✅ 新客户能自助注册 → 配置 → 嵌入Widget → 使用
- ✅ 计费和支付流程闭环
- ✅ 管理后台可查看对话和数据
- ✅ 生产环境稳定运行

## Phase 3：规模化运营（持续迭代）

### 渠道扩展
- 微信公众号接入
- 企业微信接入
- 小程序 SDK
- App SDK (React Native / Flutter)
- 抖音/快手客服对接

### 行业拓展
- 法律咨询插件
- 医疗问诊插件（注意合规）
- 电商售后插件
- 教育辅导插件
- **插件市场**：允许第三方开发者贡献插件

### 能力升级
- AI Copilot（辅助人工坐席）
- 多轮复杂任务编排（LangGraph）
- 多模态（图片理解、语音对话）
- 多语言支持
- 模型微调（基于租户数据）

### 运营工具
- A/B 测试（不同提示词对比）
- 自动质检（对话质量评分）
- 客户成功工具（健康度评分、流失预警）
- 推荐系统（根据行业推荐配置模板）

## 从八字项目迁移清单

现有代码可直接复用的部分：

| 八字项目模块 | 迁移目标 | 工作量 |
|-------------|----------|--------|
| `backend/server.py` 对话逻辑 | `core/conversation.py` + `domains/bazi/plugin.py` | 中 |
| `backend/memory_manager.py` | `memory/manager.py`（改造为多租户） | 中 |
| `backend/long_term_memory.py` | `memory/long_term.py` | 小 |
| `backend/intent_classifier.py` | `core/intent.py`（通用化） | 小 |
| `backend/slot_extractor.py` | `core/slot.py`（通用化） | 小 |
| `backend/rag.py` | `rag/retriever.py` | 小 |
| `backend/prompt_manager.py` | `domains/bazi/prompts/` | 小 |
| `frontend/js/bazi_new.js` | `domains/bazi/engine.py`（或保留前端） | 小 |
| `frontend/js/chat_widget.js` | `frontend/widget/`（Web Component重写） | 中 |
| `backend/user_manager.py` | `models/user.py` + `tenant/manager.py` | 中 |
| `backend/session_state.py` | `core/state.py` + `memory/short_term.py` | 小 |

**迁移策略：渐进式，而非推倒重来。**
1. 先搭新框架骨架
2. 逐模块迁移，每迁一个就测一个
3. 八字项目保留运行直到新平台功能完全覆盖
4. 最后切流量

## 竞品参考

| 竞品 | 定位 | 我们的差异化 |
|------|------|-------------|
| Dify | AI应用平台（通用） | 我们更聚焦客服场景，开箱即用 |
| Coze/扣子 | AI Bot平台 | 我们提供完整的客服运营工具 |
| 智齿客服 | 传统客服SaaS | 我们AI-Native，成本更低 |

**核心差异化：领域插件 + 记忆体系 + 开箱即用。**
客户不需要理解 AI，只需要上传知识文档、配置几个参数，就能拥有专业的智能客服。
``

   ├── eval/                    # 评测体系

   │   ├── benchmark.py         # 标注数据集跑分                                                                       
   │   └── ab_test.py           # A/B 测试框架                                                                         
   │                                                                                                                   
   ├── guardrails/              # 安全护栏                                                                             
   │   ├── input_filter.py      # Prompt 注入检测 + 敏感词                                                             
   │   ├── output_validator.py  # 禁用话术 + 过度承诺 + 信息泄露拦截

   │                                                                                                                   
   ├── observability/           # 可观测性
