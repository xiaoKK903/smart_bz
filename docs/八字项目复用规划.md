# 八字智能客服项目复用规划

## 1. 可直接复用的核心模块

### 1.1 记忆系统（Memory）

**模块描述**：八字智能客服的三层记忆架构是一个完整、成熟的实现，可以直接复用到通用智能客服平台。

**可复用文件**：
- `backend/memory_manager.py` - 统一记忆管理器，实现三层记忆的调度
- `backend/session_state.py` - 短期记忆（Redis），管理会话状态和槽位
- `backend/user_database.py` - 用户画像（SQLite），存储用户基本信息和八字数据
- `backend/long_term_memory.py` - 长期情景记忆（ChromaDB），存储咨询历史和情景记忆

**复用方式**：
- 作为通用智能客服平台的 `memory` 模块基础
- 需要添加租户ID字段，支持多租户隔离
- 将SQLite替换为PostgreSQL以支持更复杂的用户画像

### 1.2 核心功能模块

**模块描述**：八字智能客服实现了多个核心功能模块，可以直接复用到通用平台。

- `backend/slot_extractor.py` - 槽位提取器
- `backend/rag.py` - 检索增强生成系统

**复用方式**：
- 作为通用智能客服平台的核心引擎组件

## 2. 需要适配的模块

### 2.1 配置管理

**原文件**：`backend/config.py`

**适配需求**：
- 从单一配置改为多租户配置
- 支持每个租户独立配置模型、提示词、知识库等
- 集成环境变量和配置文件

### 2.2 服务器和API

**原文件**：`backend/server.py`

**适配需求**：
- 从简单HTTP服务器改为FastAPI
- 添加认证鉴权、限流熔断等功能
- 实现多租户路由
- 扩展API接口以支持通用平台功能

### 2.3 领域插件

**原文件**：`backend/bazi_ai_server.py`

**适配需求**：
- 从单一八字领域改为插件架构
- 实现插件加载、注册和路由机制
- 定义插件接口标准

### 2.4 数据存储

**原文件**：`backend/user_database.py`

**适配需求**：
- 从SQLite改为PostgreSQL
- 添加租户ID字段
- 设计多租户数据隔离方案

## 3. 复用实施步骤

### 3.1 准备工作
1. 创建通用智能客服平台的基础目录结构
2. 配置开发环境和依赖
3. 建立数据库和缓存服务

### 3.2 核心模块迁移
1. 复制记忆系统相关文件到 `backend/app/memory/`
2. 复制核心功能模块到 `backend/app/core/`
3. 复制工具类到 `backend/app/utils/`
4. 复制前端组件到 `frontend/widget/`

### 3.3 适配和集成
1. 修改记忆系统以支持多租户
2. 集成到FastAPI框架
3. 实现插件系统和领域路由
4. 配置多租户数据隔离

### 3.4 测试和优化
1. 运行现有测试脚本验证功能
2. 扩展测试以支持多租户场景
3. 性能测试和优化
4. 安全测试和加固

## 5. 风险评估

### 5.1 技术风险
- 数据库迁移（SQLite → PostgreSQL）可能需要额外工作
- 多租户架构的适配需要仔细设计
- 插件系统的集成需要额外开发

### 5.2 解决方案
- 制定详细的数据库迁移计划
- 设计清晰的多租户数据隔离方案
- 采用模块化设计，确保插件系统的可扩展性

八字智能客服项目提供了丰富的可复用组件，特别是其完整的记忆体系、核心功能模块和GraphRAG能力。通过合理的适配和集成，可以显著加速通用智能客服平台的开发，同时确保系统的稳定性和功能完整性。

建议按照本文档的规划，分阶段进行模块迁移和适配，优先复用核心功能，然后逐步完善插件系统和多租户支持，最终构建一个功能完整、架构清晰的通用智能客服平台。
